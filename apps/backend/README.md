# @our-haven/backend

Phase 2 backend (ADR-0004). Node + TypeScript + Fastify + OpenAPI source-of-truth + Postgres (Cloud SQL) + Firestore (`nam5`) + Cloud Run + Cloud Storage + Cloud Tasks. Pinned to GCP `us-east1` per ADR-0009.

## Quickstart

```bash
# from the repo root
npm install
cp apps/backend/.env.example apps/backend/.env
docker compose -f apps/backend/docker-compose.yml up -d
npm run backend
# → server on http://localhost:8080  (Swagger UI at /docs)
```

## Layout

```
src/
  app.ts                Fastify bootstrap + plugin registration + OpenAPI plugin
  server.ts             Process entry; dependency wiring + signal handling
  config/env.ts         Zod-validated env (US-region pinning, schema is source of truth)
  db/
    kysely.ts           Postgres connection (Kysely + pg.Pool)
    schema.ts           Kysely Database interface (filled in by 2.3 onward)
    migrations/         Kysely migrations (run via `npm run migrate:*`)
  gcp/
    firebase.ts         firebase-admin init (Auth + Firestore)
    storage.ts          GCS client + v4 signed-URL helper
    tasks.ts            Cloud Tasks client wrapper
  routes/
    health.ts           GET /v1/healthz + /v1/readyz
    uploads.ts          POST /v1/uploads/signed-url
openapi/
  openapi.yaml          Generated source-of-truth spec — DO NOT hand-edit; regenerated by scripts/emit-openapi.ts.
scripts/
  emit-openapi.ts       Boots the app, asks @fastify/swagger for the spec, writes openapi/openapi.yaml.
  check-openapi-drift.ts  CI check: fails if runtime spec ≠ checked-in yaml.
  migrate.ts            Kysely migration runner (up/down/make).
```

## Scripts

| Command | What it does |
|---|---|
| `npm run dev --workspace=@our-haven/backend` | tsx watch on src/server.ts |
| `npm run build --workspace=@our-haven/backend` | tsc → dist/ |
| `npm test --workspace=@our-haven/backend` | vitest run |
| `npm run openapi:emit --workspace=@our-haven/backend` | Regenerate openapi/openapi.yaml |
| `npm run openapi:check --workspace=@our-haven/backend` | Fail CI on spec drift |
| `npm run migrate:up --workspace=@our-haven/backend` | Apply pending migrations |
| `npm run migrate:make <name> --workspace=@our-haven/backend` | Create a new migration file |

## Deep modules

Per ADR-0004 § Consequences, the deep modules (Booking lifecycle, Pricing & Commission, Cancellation, Disintermediation detector, Search ranking, Rating reveal, Verification workflow, Retention planner) live in `packages/domain/` as pure TS — no DB/SDK imports — and get their collaborators injected at the handler layer.

## Region / residency

Hardcoded to GCP `us-east1` (fallback `us-east4`) + Firestore `nam5`. The env schema enforces this; tests will fail if anything else is set. Per ADR-0004 §5 and ADR-0009 (US-national from day one).

## Auth (placeholder)

Routes are unauthenticated in 2.1. **Ticket 2.2** introduces Firebase Auth ID-token verification via firebase-admin + a Fastify hook (`onRequest`) that decodes the bearer token, fails closed on invalid tokens, and attaches the verified subject to `req.user`. The `POST /v1/uploads/signed-url` handler's `ownerId = 'anonymous'` placeholder is wired to flip to `req.user.uid` then.
