### 1.1 — Bug: persistence shadow-merge wipes the Bookings list

**Type:** AFK · **Blocked by:** — · **Labels:** `bug` `store`

**What:** On rehydrate, `store.ts:~75` does a shallow `{ ...createSeedState(), ...persisted }`, so a stale `persisted.bookings` array fully shadows the seed. A returning user (or the client's review build) lands on an **empty Bookings page** even though the seed is full. Fix the merge so seeded + user-created bookings always show.

**Acceptance criteria:**
- [ ] A returning user with older persisted state still sees their bookings (seed merges by id; user-created rows preserved).
- [ ] No regression to user-created bookings/jobs/offers across reloads.
- [ ] Choose one: bump the persistence `KEY` (drops stale state) **or** deep/array-merge by id — document which and why in the commit.
- [ ] tsc + build + lint green.

**Files:** `app/lib/store.ts` (rehydrate merge ~L75, `KEY`).

### 1.4 — Concrete date + start–end time on the Book-request (direct request)

**Type:** AFK · **Blocked by:** — · **Labels:** `frontend` `schema` `store` · **PRD:** 24 · **Refs:** ADR-0014, CONTEXT § Offer / § Booking

**What:** A parent sending a direct Book-request picks a **concrete date + start time + end time** (not a band). Establishes the shared schedule schema on `Offer`/`Booking`, wires the composer and the booking-detail display. This is parent-specified scheduling — **not** slot-pick.

- [ ] The Book-request composer (`booking-compose` and the `MessageThread` Book-request) exposes a date picker + start/end time pickers.
- [ ] The chosen date + time render on booking-detail and in the sent Offer bubble.
- [ ] Caregiver can still Accept / Counter / Decline; no caregiver bookable calendar introduced.
- [ ] tsc + build + lint green.

**Files:** `app/lib/types.ts` (`Offer`, `Booking`, `UIState.composeDraft`), `app/lib/store.ts` (`createBookingFromDraft`, offer creation), `app/screens/booking-compose/page.tsx` + `web.tsx`, `app/components/MessageThread.tsx`, `app/screens/booking-detail/page.tsx` + `web.tsx`.

### 1.7 — Booking Series: entity + award materialisation + hub grouping

**Type:** AFK · **Blocked by:** 1.6 · **Labels:** `schema` `store` `frontend` · **PRD:** 34a, 90 · **Refs:** ADR-0014, CONTEXT § Booking

**What:** Awarding a recurring Job creates a **Booking Series** that materialises **all** occurrence Bookings up front, each a normal per-session Booking grouped under the Series. The Bookings list renders the Series as one group row that expands to its occurrences.

**Acceptance criteria:**
- [ ] `BookingSeries` entity + nullable `seriesId` on `Booking` in `types.ts`; a demo series in `seed.ts`.
- [ ] `awardJob` (recurring) materialises every occurrence Booking up front with the Agreed Rate per occurrence; one-off award still creates a single Booking.

- [ ] Cancelling one occurrence leaves the rest; cancelling the Series cancels still-upcoming occurrences.
- [ ] tsc + build + lint green.

**Files:** `app/lib/types.ts` (`BookingSeries`, `Booking.seriesId`), `app/lib/store.ts` (`awardJob`, series selectors), `app/lib/seed.ts`, `app/screens/bookings/page.tsx` + `web.tsx`, `app/screens/jobs-detail/*` (recurring award).

### 1.8 — Restore confirm-hours reconciliation (proposed hours + Confirm/Dispute)

**Type:** AFK · **Blocked by:** — · **Labels:** `frontend` `store` · **PRD:** 32, 38, 54 · **Refs:** ADR-0013, CONTEXT § Session / § Dispute

**What:** Undo the prototype's flattening ("nothing to reconcile; agreed total is final"). At session end the caregiver proposes **actual hours** (overtime or short); the parent gets the ~24h review window to **Confirm & pay** the proposed amount **or Dispute** (→ `disputed`), auto-confirm if untouched.

**Acceptance criteria:**

- [ ] Dispute action moves the booking to `disputed` (new/used status) and routes it to the Disputes view.
- [ ] Remove the "There's nothing to reconcile / amount agreed at booking is final" copy.
- [ ] tsc + build + lint green.

**Files:** `app/screens/booking-detail/page.tsx` + `web.tsx` (`ConfirmHoursSheet`), `app/lib/store.ts` (`endSession`/`proposeHours`/`disputeBooking`, `BookingStatus`), `app/lib/types.ts` (status), `app/screens/provider-session/*`.
